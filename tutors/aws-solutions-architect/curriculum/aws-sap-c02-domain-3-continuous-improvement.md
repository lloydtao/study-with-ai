# Domain 3: Continuous Improvement for Existing Solutions

Weight: 25%

## Unit goal

Build the ability to assess live AWS workloads, identify meaningful improvements, and prioritize changes based on measurable tradeoffs.

## Learning objectives

| Objective ID | Learning objective | Focus areas | Recommended references |
| --- | --- | --- | --- |
| SAP-D3-O1 | Assess an existing workload against AWS Well-Architected principles and identify the highest-value gaps. | Review methodology, risk identification, pillar-based analysis, remediation prioritization | AWS Well-Architected Framework, SAP-C02 exam guide, AWS Architecture Center |
| SAP-D3-O2 | Improve reliability and operability by strengthening observability, automation, and incident response patterns. | Metrics, logs, tracing, alarms, runbooks, game days, operational readiness | AWS Well-Architected Framework, AWS Architecture Center, Solutions Architect ramp-up guide |
| SAP-D3-O3 | Improve performance and cost efficiency based on actual workload behavior rather than assumptions. | Rightsizing, caching, scaling patterns, data tiering, storage lifecycle, cost-performance tradeoffs | AWS Well-Architected Framework, AWS Architecture Center, SAP-C02 in-scope services page |
| SAP-D3-O4 | Improve security posture in existing environments without destabilizing critical workloads. | Least privilege, key management, network hardening, detection, patching, secrets handling | AWS Well-Architected Framework, AWS Prescriptive Guidance, SAP-C02 exam guide |
| SAP-D3-O5 | Build a practical modernization backlog that sequences improvements by risk, dependency, and business value. | Technical debt, remediation waves, stakeholder communication, incremental change planning | AWS Prescriptive Guidance, SAP-C02 exam guide, Solutions Architect ramp-up guide |

## Typical mastery signals

- Can distinguish urgent remediation from lower-value optimization work.
- Can explain how to improve a system iteratively without treating redesign as the only answer.
- Can tie recommendations to measurable outcomes such as recovery time, latency, error rates, or spend.
