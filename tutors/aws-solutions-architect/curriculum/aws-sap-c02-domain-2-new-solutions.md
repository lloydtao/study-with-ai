# Domain 2: Design for New Solutions

Weight: 29%

## Unit goal

Build the ability to design net-new AWS solutions that satisfy business, technical, security, resilience, and cost constraints from the start.

## Learning objectives

| Objective ID | Learning objective | Focus areas | Recommended references |
| --- | --- | --- | --- |
| SAP-D2-O1 | Select compute and application integration patterns that match workload scale, operational model, and latency requirements. | EC2, containers, serverless, event-driven design, asynchronous integration, orchestration | SAP-C02 exam guide, SAP-C02 in-scope services page, AWS Architecture Center |
| SAP-D2-O2 | Choose storage and database designs that balance performance, consistency, durability, recovery objectives, and cost. | Relational vs NoSQL tradeoffs, object vs block vs file storage, replication, caching, backups | SAP-C02 exam guide, AWS Architecture Center, AWS Well-Architected Framework |
| SAP-D2-O3 | Design network architectures for secure, scalable connectivity across users, systems, and Regions. | VPC layout, ingress and egress, private access patterns, hybrid connectivity, global traffic routing | SAP-C02 in-scope services page, AWS Architecture Center, AWS Well-Architected Framework |
| SAP-D2-O4 | Design resilient architectures that meet availability and disaster recovery targets without overbuilding. | Multi-AZ vs multi-Region, failure isolation, backup and restore, pilot light, warm standby, active-active | SAP-C02 exam guide, AWS Well-Architected Framework, AWS Architecture Center |
| SAP-D2-O5 | Evaluate end-to-end design tradeoffs using AWS Well-Architected thinking. | Security, reliability, performance, operational excellence, sustainability, cost optimization | AWS Well-Architected Framework, SAP-C02 exam guide, Solutions Architect ramp-up guide |

## Typical mastery signals

- Can justify service selection with explicit tradeoffs instead of listing features.
- Can connect business requirements to architecture qualities such as durability, latency, and operational effort.
- Can compare resilience patterns and explain why a simpler option may be preferable.
