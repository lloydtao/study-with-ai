# Domain 4: Accelerate Workload Migration and Modernization

Weight: 20%

## Unit goal

Build the ability to choose migration and modernization strategies that reduce risk, preserve business continuity, and move legacy workloads toward better target-state architectures.

## Learning objectives

| Objective ID | Learning objective | Focus areas | Recommended references |
| --- | --- | --- | --- |
| SAP-D4-O1 | Assess application portfolios and choose an appropriate migration strategy using the 7 Rs. | Retire, retain, rehost, relocate, repurchase, replatform, refactor | Large migration strategies guide, AWS Prescriptive Guidance, SAP-C02 exam guide |
| SAP-D4-O2 | Plan migration waves that account for dependency mapping, shared services, and cutover risk. | Discovery, wave planning, landing zones, sequencing, rollback considerations | AWS Prescriptive Guidance, AWS Architecture Center, SAP-C02 exam guide |
| SAP-D4-O3 | Design data migration and replication approaches that meet downtime, consistency, and rollback requirements. | Online vs offline migration, replication, cutover patterns, dual-write risks, validation | AWS Prescriptive Guidance, SAP-C02 in-scope services page, AWS Architecture Center |
| SAP-D4-O4 | Choose modernization patterns that improve agility without forcing unnecessary full rewrites. | Strangler pattern, incremental decomposition, managed service adoption, platform standardization | AWS Prescriptive Guidance, AWS Architecture Center, Solutions Architect ramp-up guide |
| SAP-D4-O5 | Validate post-migration success and identify the next optimization steps after landing on AWS. | Operational readiness, cost review, performance validation, resilience checks, backlog creation | AWS Prescriptive Guidance, AWS Well-Architected Framework, SAP-C02 exam guide |

## Typical mastery signals

- Can explain why the same business context can justify different migration strategies for different applications.
- Can identify where data movement and cutover choices create the highest risk.
- Can describe modernization as a staged journey rather than a one-step rewrite.
